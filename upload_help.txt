Install a WASM file to the ledger without creating a contract instance

Usage: stellar.exe contract upload [OPTIONS] --source-account <SOURCE_ACCOUNT> --wasm <WASM>

Options:
  -s, --source-account <SOURCE_ACCOUNT>
          Account that where transaction originates from. Alias `source`. Can be an identity (--source alice), a public key (--source GDKW...), a muxed account (--source MDAΓÇª), a secret key (--source SC36ΓÇª), or a seed phrase (--source "kite urbanΓÇª"). If `--build-only` or `--sim-only` flags were NOT provided, this key will also be used to sign the final transaction. In that case, trying to sign with public key will fail [env: STELLAR_ACCOUNT=] [aliases: --source]
      --sign-with-key <SIGN_WITH_KEY>
          Sign with a local key or key saved in OS secure storage. Can be an identity (--sign-with-key alice), a secret key (--sign-with-key SC36ΓÇª), or a seed phrase (--sign-with-key "kite urbanΓÇª"). If using seed phrase, `--hd-path` defaults to the `0` path [env: STELLAR_SIGN_WITH_KEY=]
      --hd-path <HD_PATH>
          If using a seed phrase to sign, sets which hierarchical deterministic path to use, e.g. `m/44'/148'/{hd_path}`. Example: `--hd-path 1`. Default: `0`
      --sign-with-lab
          Sign with https://lab.stellar.org [env: STELLAR_SIGN_WITH_LAB=]
      --sign-with-ledger
          Sign with a ledger wallet [env: STELLAR_SIGN_WITH_LEDGER=]
      --wasm <WASM>
          Path to wasm binary
  -i, --ignore-checks
          Whether to ignore safety checks when deploying contracts
  -h, --help
          Print help

Options (RPC):
      --rpc-url <RPC_URL>
          RPC server endpoint [env: STELLAR_RPC_URL=]
      --rpc-header <RPC_HEADERS>
          RPC Header(s) to include in requests to the RPC provider, example: "X-API-Key: abc123". Multiple headers can be added by passing the option multiple times [env: STELLAR_RPC_HEADERS=]
      --network-passphrase <NETWORK_PASSPHRASE>
          Network passphrase to sign the transaction sent to the rpc server [env: STELLAR_NETWORK_PASSPHRASE=]
  -n, --network <NETWORK>
          Name of network to use from config [env: STELLAR_NETWORK=]
      --fee <FEE>
          fee amount for transaction, in stroops. 1 stroop = 0.0000001 xlm [env: STELLAR_FEE=] [default: 100]
      --cost
          Output the cost execution to stderr
      --instructions <INSTRUCTIONS>
          Number of instructions to simulate
      --instruction-leeway <INSTRUCTION_LEEWAY>
          Allow this many extra instructions when budgeting resources during transaction simulation
      --build-only
          Build the transaction and only write the base64 xdr to stdout
